## What This Template Does

This template automates **DeFi vault harvesting** — a specialized version of the keeper pattern for vault-specific operations like yield harvesting and compounding.

The workflow:

1. **Cron trigger** fires every 5 minutes (configurable)
2. **Reads** vault state: `shouldHarvest()`, `pendingYield()`, `totalHarvested()`, `harvestCount()`
3. **If profitable** (time elapsed AND yield > threshold): Sends a signed report to trigger harvest on-chain
4. **If not profitable**: Logs and skips — no wasted gas on unprofitable harvests
5. **On-chain**: Harvests pending yield, resets state, and emits an event

This pattern is compatible with vault protocols like Beefy, Yearn, and Alchemix.

**Key Technologies:**

- **CRE (Chainlink Runtime Environment)** — Orchestrates periodic workflows with DON consensus
- **Profitability checking** — Time + yield threshold validation off-chain
- **Yield tracking** — Harvest counter and history

## Getting Started

<Accordion title="Install dependencies" number={1}>

```bash
cd my-workflow && bun install && cd .. && cd contracts && bun install && cd ..
```

</Accordion>

<Accordion title="Run tests" number={2}>

```bash
cd my-workflow && bun test
```

</Accordion>

<Accordion title="Simulate the workflow" number={3}>

Dry run:

```bash
cre workflow simulate my-workflow --target staging-settings
```

With broadcast (writes on-chain):

```bash
cre workflow simulate my-workflow --target staging-settings --broadcast
```

</Accordion>

<Accordion title="Deploy your own contract" number={4}>

Deploy `VaultHarvester` with constructor arguments: forwarder address, harvest interval in seconds, and minimum yield threshold in wei:

```bash
forge create src/VaultHarvester.sol:VaultHarvester \
  --broadcast --private-key <KEY> --rpc-url <RPC> \
  --constructor-args 0x15fc6ae953e024d975e77382eeec56a9101f9f88 300 1000000000000000000
```

Simulate yield accrual for testing:

```bash
cast send <YOUR_CONTRACT> "accrueYield(uint256)" 2000000000000000000
```

Then update `contractAddress` in `my-workflow/config.staging.json`.

</Accordion>

## Customization

- **Change schedule** — Edit `schedule` in `config.staging.json` (cron syntax)
- **Change profitability check** — Modify `shouldHarvest()` in the contract
- **Change harvest logic** — Update `_processReport()` for your vault's harvest + compound strategy
