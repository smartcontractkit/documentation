This tutorial will guide you through enabling your tokens in CCIP using only your web browser and [Remix IDE](https://remix.ethereum.org). You'll learn how to deploy tokens and set up token pools without needing to install any development tools.

## Before You Begin

### Prerequisites

1. **Web Browser Setup**:

   - Install [MetaMask](https://metamask.io/) browser extension
   - Configure MetaMask with your blockchains. You can do this in two ways:

     **Option 1: Using Chainlist**

     1. Visit [Chainlist](https://chainlist.org/)
     1. Search for your desired blockchains (check "Include Testnets" for testnets)
     1. Click "Add to MetaMask" for each blockchain
     1. Confirm the addition in MetaMask popup

     **Option 2: Manual Configuration**

     - Follow MetaMask's [official guide](https://support.metamask.io/networks-and-sidechains/managing-networks/how-to-add-a-custom-network-rpc/) to add blockchains manually

1. Get native tokens for gas fees:
   - For testnets, you can get these from blockchain-specific faucets
   - For mainnets, you'll need to acquire the native tokens through an exchange

<Aside type="note" title="Blockchain Requirements">
  Make sure your chosen blockchains are CCIP-supported. You can find the list of supported blockchains in the [CCIP
  Directory](/ccip/directory).
</Aside>

### Source Blockchain Setup

#### Deploy Token

1. Ensure MetaMask is connected to the source blockchain: <ChainValue type="source" bold required={false} client:load />

1. In the "Deploy & Run Transactions" tab:
   - Set Environment to "Injected Provider - MetaMask"
   - Select **BurnMintERC677** contract
   - Configure deployment parameters:
     - `name`: Your token's full name (e.g., "My Cross Chain Token")
     - `symbol`: A short identifier for your token (e.g., "MCCT")
     - `decimals`: Number of decimal places for token amounts (e.g., 18)
     - `maxSupply`: Maximum total supply in wei (use 0 for unlimited supply)
   - Click "Deploy" and confirm in MetaMask
   - Your new token address will be displayed in the "Deployed Contracts" section
   - Save your token address:<br/>
     <div style={{ marginTop: "0.5rem" }}>
       <ContractAddress type="token" chain="source" placeholder="Enter deployed token address" client:load />
     </div>

<Aside type="note" title="About the Parameters">
  The `name` and `symbol` help identify your token in wallets and applications - Using 18 `decimals` is standard for
  most ERC20 tokens (1 token = 1000000000000000000 wei) - Setting `maxSupply` to 0 allows unlimited minting. For a
  limited supply, specify the amount in wei (e.g., for max 1000 tokens with 18 decimals: 1000000000000000000000)
</Aside>

#### Claim and Accept Admin Role

1. Ensure MetaMask is still connected to the source blockchain: <ChainValue type="source" bold required={false} client:load />

1. Register as admin:

   - In the "Deploy & Run Transactions" tab, select **RegistryModuleOwnerCustom** contract
   - Use "At Address" with the RegistryModuleOwnerCustom contract: <div style={{ display: "inline" }}><NetworkAddress type="registryModule" chain="source" required={false} client:only="react" /></div>
   - The RegistryModuleOwnerCustom will be displayed in the "Deployed Contracts" section
   - Click on the RegistryModuleOwnerCustom contract address to open the contract details
   - Call `registerAdminViaOwner` with your token address: <div style={{ display: "inline" }}><StoredContractAddress type="token" required={false} chain="source" client:only="react" /></div>
   - Confirm the transaction in MetaMask

1. Accept the admin role:
   - In the "Deploy & Run Transactions" tab, select **TokenAdminRegistry** contract
   - Use "At Address" with the TokenAdminRegistry contract: <div style={{ display: "inline" }}><NetworkAddress type="tokenAdminRegistry" chain="source" required={false} client:only="react" /></div>
   - The TokenAdminRegistry will be displayed in the "Deployed Contracts" section
   - Click on the TokenAdminRegistry contract address to open the contract details
   - Call `acceptAdmin` with your token address: <div style={{ display: "inline" }}><StoredContractAddress type="token" required={false} chain="source" client:only="react" /></div>
   - Confirm the transaction in MetaMask

### Configure Cross-Chain Communication

#### Source Blockchain Configuration

Ensure MetaMask is still connected to the source blockchain: <ChainValue type="source" bold required={false} client:load />

##### Grant Burn and Mint Privileges

<Aside type="note">Skip this section if you deployed a LockReleaseTokenPool</Aside>

Grant privileges to your token pool:

- In the list of deployed contracts, select your **BurnMintERC677** token at <StoredContractAddress type="token" chain="source" required={false} client:only="react" />
- Call `grantMintAndBurnRoles` with your token pool address: <StoredContractAddress type="tokenPool" chain="source" required={false} client:only="react" />
- Confirm the transaction in MetaMask

##### Configure Remote Pool

Enable the destination pool:

- In the list of deployed contracts, select your token pool at <StoredContractAddress type="tokenPool" chain="source" required={false} client:only="react" />
- Call `addRemotePool` with:
  - The destination blockchain selector: `remoteChainSelector` <NetworkAddress type="chainSelector" chain="destination" required={false} client:only="react" />
  - The destination pool address: `remotePoolAddress` <StoredContractAddress type="tokenPool" chain="destination" required={false} encode client:only="react" />
- Confirm the transaction in MetaMask
