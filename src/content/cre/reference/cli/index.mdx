The CRE Command Line Interface (CLI) is your primary tool for developing, testing, deploying, and managing workflows. It handles project setup, contract binding generation (Go workflows only), local simulation, and workflow lifecycle management.

## Global flags

These flags can be used with any `cre` command.

| <div style="width: 150px;">Flag</div> | Description                                                                                                                                                                                |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `-h, --help`                          | Displays help information for any command                                                                                                                                                  |
| `-e, --env`                           | Specifies the path to your `.env` file (default: `".env"`)                                                                                                                                 |
| `-T, --target`                        | Sets the target environment from your configuration files                                                                                                                                  |
| `-R, --project-root`                  | Specifies the path to the project root directory. By default, the CLI automatically finds the project root by searching for `project.yaml` in the current directory and parent directories |
| `-v, --verbose`                       | Enables verbose logging to print `DEBUG` level logs                                                                                                                                        |

## Commands overview

### Authentication

Manage your authentication and account credentials.

- **`cre login`** ‚Äî Authenticate with the CRE UI, save credentials locally, and cache tenant context (`context.yaml`)
- **`cre logout`** ‚Äî Revoke authentication tokens and remove local credentials and cached tenant context
- **`cre whoami`** ‚Äî Show your current account details

[View authentication commands ‚Üí](/cre/reference/cli/authentication)

---

### Project Setup

Initialize projects and generate contract bindings (Go only).

- [**`cre init`**](/cre/reference/cli/project-setup) ‚Äî Initialize a new CRE project (interactive wizard or **`--non-interactive`** with flags for CI/CD)
- **`cre generate-bindings`** (Go only) ‚Äî Generate Go bindings from contract ABI files for type-safe contract interactions

[View project setup commands ‚Üí](/cre/reference/cli/project-setup)

---

### Account Management

Manage your linked public key addresses and deployment access.

- **`cre account access`** ‚Äî Check deployment access status or submit an Early Access request
- **`cre account link-key`** ‚Äî Link a public key address for public onchain registry operations
- **`cre account list-key`** ‚Äî List workflow owners linked to your organization
- **`cre account unlink-key`** ‚Äî Unlink a public key address from your account

[View account management commands ‚Üí](/cre/reference/cli/account)

---

### Workflow Commands

Manage workflows throughout their entire lifecycle.

- **`cre workflow build`** ‚Äî Compile a workflow to a WASM binary (without uploading or deploying)
- **`cre workflow simulate`** ‚Äî Compile and execute workflows in a local simulation environment
- **`cre workflow hash`** ‚Äî Display content hashes for a workflow's binary and config (matches the workflow ID)
- **`cre workflow deploy`** ‚Äî Deploy a workflow to the registry selected by `deployment-registry`
- **`cre workflow activate`** ‚Äî Activate a workflow in its configured registry
- **`cre workflow pause`** ‚Äî Pause a workflow in its configured registry
- **`cre workflow delete`** ‚Äî Delete all versions of a workflow from its configured registry
- **`cre workflow custom-build`** ‚Äî Convert a workflow to use a custom self-compiled WASM build

[View workflow commands ‚Üí](/cre/reference/cli/workflow)

---

### Secrets Management

Manage secrets stored in the Vault DON for use in your workflows.

- **`cre secrets create`** ‚Äî Create new secrets from a YAML file
- **`cre secrets update`** ‚Äî Update existing secrets
- **`cre secrets delete`** ‚Äî Delete secrets
- **`cre secrets list`** ‚Äî List secret identifiers in a namespace

[View secrets management commands ‚Üí](/cre/reference/cli/secrets)

---

### Registry Commands

Inspect the workflow registries available to your organization. A workflow registry is the system that tracks which workflows are registered to your organization and makes them available to the Chainlink DON for execution.

- **`cre registry list`** ‚Äî Display the registries configured for your organization, including type and onchain address

[View registry commands ‚Üí](/cre/reference/cli/registry)

---

### Template Sources

Manage the GitHub repositories that `cre init` uses to discover templates.

- **`cre templates list`** ‚Äî List templates (human output includes required networks; **`--json`** for scripts)
- **`cre templates add`** ‚Äî Add a custom or third-party template repository
- **`cre templates remove`** ‚Äî Remove a template repository source

[View template source commands ‚Üí](/cre/reference/cli/templates)

---

### Utilities

Additional utility commands.

- **`cre update`** ‚Äî Update the CRE CLI to the latest version
- **`cre version`** ‚Äî Print the current version of the CRE CLI

[View utility commands ‚Üí](/cre/reference/cli/utilities)

## Typical development workflow

The typical workflow development process uses these commands in sequence:

1. **`cre init`** ‚Äî Initialize your project
1. **`cre generate-bindings`** ‚Äî Generate contract bindings from ABIs (Go workflows only, if interacting with contracts)
1. **`cre workflow simulate`** ‚Äî Test your workflow locally
1. **`cre workflow build`** ‚Äî (Optional) Compile independently; useful in CI/CD pipelines
1. **`cre workflow hash`** ‚Äî (Optional) Inspect content hashes before deploying
1. **`cre workflow deploy`** ‚Äî Deploy your workflow to the registry
1. **`cre workflow activate`** / **`cre workflow pause`** ‚Äî Control workflow execution
