`solana.Client` is the Go SDK's interface for writing data to Solana programs. It submits cryptographically signed reports through the Keystone Forwarder program, which verifies DON signatures and CPIs into your receiver program.

<Aside type="note" title="Write only — for now">
  The current release supports **onchain writes** only. Solana Read and Solana Log Trigger capabilities are in active
  development and will be documented when available.
</Aside>

## How it works

The Go SDK uses **generated Anchor bindings** for Solana program interactions. You provide your program's Anchor IDL, run the binding generator, and the SDK creates typed Go structs with `WriteReportFrom<StructName>()` methods that handle Borsh encoding, account hashing, and report submission.

Key aspects of the Go approach:

- Anchor IDL-based code generation via `cre generate-bindings solana`
- `github.com/gagliardetto/binary` handles Borsh serialization inside generated bindings
- `bindings.CalculateAccountsHash` and `bindings.ForwarderReport` are the core primitives
- Account public keys are 32-byte slices; your workflow populates `[]*solana.AccountMeta` directly
